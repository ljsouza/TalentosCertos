01 — Panorama

Onde o produto está hoje

O protótipo já cobre um ecossistema completo de empregabilidade regional — bem além de um quadro de vagas. São cinco frentes de produto, todas navegáveis.

★

Posicionamento: não é só um classificado de vagas. É um hub regional de talento que combina recrutamento tradicional, conteúdo editorial, economia de criadores, trabalho sob demanda e novos modelos de contratação por mérito e por experiência.

Inventário de funcionalidades

Vagas núcleo do produto
  • Busca e filtros (área, cidade, ordenação por recência/salário)
  • Anúncio com modalidade (presencial, híbrido, remoto)
  • Prazo de divulgação com encerramento automático + alerta ao gestor
  • Compartilhamento (WhatsApp, link, Instagram, Facebook, X)
  • Candidatura e vagas salvas
Área da empresa
  • Login, painel e perfil institucional (logo, fundação, porte, ramo)
  • Publicação de vaga assistida por IA (descrição, requisitos, faixa, triagem, anúncio)
  • Minhas vagas: cancelamento com motivo, vínculo a validade/status
  • Ver candidatos, seleção e mensagens-padrão por WhatsApp
  • Conta verificada
Área do candidato
  • Cadastro com currículo/perfil e login próprio
  • Radar de vagas por WhatsApp com consentimento LGPD e máscara de telefone
  • Menu de conta (avatar, editar cadastro, sair)
Administração
  • Gestão de usuários (empresas e seus colaboradores)
  • Aprovação de conteúdo e moderação
  • Cadastro de empresas e vínculo a pacotes
  • Pacotes comerciais: Atrair, Conectar, CrescerPro (limites de vaga/mês)
Conteúdo & comunidade
  • Carreira & RH — portal editorial com reportagens
  • Tribuna do Talento — creator economy: profissionais publicam e ganham destaque
Novos mercados
  • Na Hora — marketplace de profissionais temporários sob demanda
  • Talento 50+ — banco de talentos de profissionais sêniores
  • Desafios — recrutamento por competência com avaliação e ranking por IA

Tudo isso existe hoje como protótipo de alta fidelidade (front-end). O passo seguinte é dar a ele um back-end real — dados persistentes, autenticação, pagamentos e IA — com o menor custo possível no início.

02 — Ferramentas

A stack gratuita recomendada

O objetivo é começar com custo R$ 0/mês e só passar a pagar quando o uso (e a receita) justificarem. As escolhas abaixo priorizam um caminho único, integrado e fácil de manter.

Banco de dados

SupabaseRecomendadoFree tier

PostgreSQL gerenciado com muito mais que banco: já traz autenticação (e-mail, Google, etc.), armazenamento de arquivos (logos, currículos), APIs automáticas, realtime e funções serverless. Para uma plataforma relacional como esta — vagas, empresas, candidatos, pacotes, desafios — o modelo SQL é o ideal, e o pacote elimina a necessidade de montar back-end do zero.

500 MB DB1 GB storage50k usuários ativos/mêsAuth + RLS incluídos

Alternativas válidas

NeonPostgres serverless, free tier generoso e escala a zero. Ótimo se quiser separar o banco do resto. Não inclui auth/storage.
Cloudflare D1SQLite na borda, gratuito e rápido. Combina com hospedagem na Cloudflare, mas é mais novo/limitado.
MongoDB Atlas512 MB grátis. Só se preferir NoSQL — menos natural para dados muito relacionais.
FirebaseBom para realtime e protótipos, porém o modelo de dados e custos ao escalar fogem do relacional.

Ambiente para rodar o SaaS

VercelRecomendadoHobby grátis

Hospedagem feita sob medida para o front-end React do projeto. Deploy automático a cada push no Git, HTTPS e CDN global inclusos, e funções serverless para a lógica de back-end que não ficar no Supabase (ex.: integrações, webhooks de pagamento, chamadas de IA).

Deploy contínuoHTTPS + CDNServerless functionsDomínio próprio

Alternativas válidas

NetlifyPraticamente equivalente à Vercel para este caso. Escolha por preferência.
Cloudflare Pages + WorkersLimites gratuitos muito generosos. Stack completa (Pages + Workers + D1 + R2) se quiser tudo na Cloudflare.
✓

Combinação base sugerida: front-end React na Vercel + dados, login e arquivos no Supabase. Dois serviços, ambos com plano gratuito real, cobrindo 100% do MVP sem cartão de crédito.

Pagamentos não têm "plano grátis" no sentido de mensalidade, mas não cobram nada enquanto não há vendas — você paga uma porcentagem só quando recebe.

03 — Roadmap

Construção por fases

Do essencial ao disruptivo. Cada fase entrega valor real sozinha e prepara o terreno para a próxima. As funcionalidades mais complexas — IA, marketplaces, pagamentos — vêm depois que a base estiver firme e gerando tração.

Fase 0

Fundação

Base técnica

Montar o esqueleto: infraestrutura, contas e modelo de dados. Nada visível ao usuário, tudo indispensável.

01Criar projetos na Vercel e no Supabase, conectar ao Git
02Modelar o banco: empresas, usuários, vagas, candidaturas, pacotes
03Migrar o protótipo React para projeto com build (Vite)
04Definir papéis e permissões (RLS): candidato, empresa, admin
SupabaseVercelModelagem SQL

04Talento 50+: banco sênior com formatos de colaboração

Por que esta ordem? As Fases 0–1 provam o produto e podem ir ao ar sem custo. A Fase 2 liga a receita antes de gastar mais. A Fase 3 traz tráfego barato via conteúdo. Só então a Fase 4 investe nos diferenciais caros e complexos — com usuários, dados e caixa para sustentá-los.
