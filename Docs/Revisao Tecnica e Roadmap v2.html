02 — Ferramentas

A stack gratuita recomendada

Começar com custo R$ 0/mês e só pagar quando o uso (e a receita) justificarem. As escolhas priorizam um caminho único, integrado e fácil de manter.

Banco de dados

SupabaseRecomendadoFree tier

PostgreSQL gerenciado com muito mais que banco: autenticação pronta, storage de arquivos (logos, currículos), APIs automáticas, realtime, funções serverless e pgvector — extensão de busca semântica essencial para os recursos de IA (match candidato↔vaga). Para dados relacionais como vagas, empresas, candidatos e pacotes, o modelo SQL é o ideal.

500 MB DB1 GB storage50k MAU authpgvector incluído

Alternativas válidas

NeonPostgres serverless com free tier generoso e escala a zero. Ótimo se quiser separar o banco; não inclui auth/storage.
Cloudflare D1SQLite na borda, gratuito e rápido. Combina com hospedagem Cloudflare; mais novo/limitado.
TursoSQLite distribuído (libSQL), free tier amplo e baixa latência regional.
MongoDB Atlas512 MB grátis. Só se preferir NoSQL — menos natural para dados muito relacionais.

Ambiente para rodar a aplicação

VercelRecomendadoHobby grátis

Hospedagem sob medida para o front-end React do projeto. Deploy automático a cada push no Git, HTTPS e CDN global, e funções serverless para a lógica que não ficar no Supabase (integrações, webhooks de pagamento, chamadas de IA). Suporta rewrites de caminho — peça-chave para servir em /empregos (seção 04).

Deploy contínuoHTTPS + CDNServerless / EdgeRewrites de path

Alternativas válidas

NetlifyPraticamente equivalente à Vercel para este caso. Escolha por preferência.
Cloudflare Pages + WorkersLimites gratuitos muito generosos; stack completa (Pages + Workers + D1 + R2) se quiser tudo na Cloudflare.
✓

Combinação base: front-end React na Vercel + dados, login, arquivos e busca semântica no Supabase. Dois serviços, ambos com plano gratuito real, cobrindo 100% do MVP sem cartão de crédito.

03 — Inteligência

As IAs embarcadas na plataforma

A inteligência não é um recurso único — é uma camada que cresce a cada versão. Abaixo, o catálogo de IAs previstas, o que cada uma faz e o modelo gratuito que pode sustentá-la no início.

01IA Cria Vaga

Gera descrição, requisitos, faixa salarial, pergunta de triagem e texto do anúncio a partir de um briefing curto da empresa.

Modelo: Gemini Flash / Groq (Llama) · generativo
02Match candidato ↔ vaga

Score de compatibilidade por similaridade semântica entre perfil e vaga; alimenta o anel de match e o radar.

Tech: embeddings + pgvector · recuperação
03Extração de currículo

Lê PDF ou LinkedIn e estrutura o perfil (experiências, skills, formação) automaticamente no cadastro.

Modelo: Gemini (visão/texto) · extração
04Triagem por Resposta Ativa

Transcreve a resposta de 30s (áudio/vídeo) e avalia clareza e aderência, filtrando candidaturas em massa.

Modelo: Whisper + LLM · transcrição+análise
05Ranking de Desafios

Avalia entregas de provas práticas e ordena candidatos por competência demonstrada, com justificativa.

Modelo: LLM com rubrica · avaliação
06Score de intencionalidade

Estima o interesse real do candidato a partir de sinais de comportamento, priorizando quem tem maior intenção.

Tech: features + LLM · classificação
07Curadoria editorial

Apoia a moderação da Tribuna/Carreira & RH, sugere pautas e destaca automaticamente conteúdo que viraliza.

Modelo: LLM + métricas · moderação
08Assistente de Inclusão (PCD)

Cruza adaptações necessárias do candidato com a acessibilidade declarada da vaga e sugere texto inclusivo.

Modelo: LLM + regras · recomendação
09Copiloto de carreira

Chat que orienta o candidato (currículo, faixa salarial, preparo) usando os dados reais do mercado regional.

Modelo: LLM + RAG (pgvector) · conversacional
10Agente de recrutamento

Visão futura: conduz a busca ativa, pré-seleciona e agenda — o recrutador só aprova. O ápice da inteligência.

Modelo: LLM + ferramentas (agente) · autônomo
✦

Estratégia de custo da IA: começar com cotas gratuitas (Gemini Flash, Groq) para validar; usar embeddings + pgvector para o match (barato e roda no próprio banco); e só migrar para modelos pagos/maiores nos recursos que comprovarem retorno. Toda chamada de IA passa por uma função serverless — fácil de trocar de provedor sem mexer no front-end.

04 — Integração

Acoplar ao maringapost.com.br/empregos

A plataforma deve viver dentro do domínio principal — em /empregos — para que toda a navegação compute tempo de uso e audiência no mesmo domínio do jornal, sem fragmentar métricas nem autoridade de SEO.

maringapost.com.br/empregos

Por que subdiretório, e não subdomínio

  • Mesma origem — /empregos é o mesmo host: cookies, sessão e analytics são compartilhados sem configuração de cross-domain.
  • Tempo de uso somado — a sessão do leitor que vem do jornal continua na mesma propriedade; o tempo em /empregos entra no total do domínio.
  • SEO herdado — um subdiretório herda a autoridade do domínio principal; um subdomínio (empregos.maringapost...) é tratado como site à parte.
  • Marca única — o usuário nunca "sai" do MaringáPost.

Como servir o app em /empregos

Roteamento
Reverse proxy (recomendado): o servidor/edge do maringapost.com.br encaminha tudo sob /empregos/* para a aplicação na Vercel. Alternativa: rewrites da Vercel/Cloudflare se o site principal já estiver lá.
Front-end
App React buildado com base path /empregos (router com basename e assets sob esse prefixo), para que links e recursos resolvam corretamente.
API & dados
Funções serverless sob /empregos/api/* e Supabase como back-end — todas as chamadas saem do mesmo host.
Analytics
Uma única propriedade GA4/Matomo do domínio; o app dispara page views de SPA em cada troca de tela. Tempo de uso e funil ficam consolidados com o resto do jornal.
Sessão / SSO
Cookies de primeira parte no domínio raiz permitem, no futuro, login único entre jornal e empregos.
→

Resumo: hospede o app na Vercel, mas exponha-o em maringapost.com.br/empregos via reverse proxy/rewrite. Mesma origem = mesma audiência, mesmo SEO, mesma conta de analytics. O leitor flui do jornal para as vagas sem perceber a troca — e o tempo de navegação soma para o domínio principal.

Por que esta ordem? v0–v1 provam o produto sem custo. v1.5 injeta a primeira IA de alto impacto. v2 liga a receita e o match. v2.5 traz tráfego barato via conteúdo. Cada versão sobe a barra de inteligência sem nunca quebrar o que já funciona.
